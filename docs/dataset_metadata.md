# Dataset README / Summary

## Context

Breast cancer clinical summaries are increasingly generated by Large Language Models (LLMs) to assist with surgical planning, tumor board preparation, and multidisciplinary documentation. However, LLM-generated summaries may contain **fabricated** (hallucinated) clinical features not present in the source documents, or may **omit** critical features that are present. These errors pose patient safety risks.

## Objective

Evaluate the accuracy, omission rate, and fabrication rate of LLM-extracted clinical features compared to human annotator extraction, using scanned radiology and pathology reports as ground truth. The study aims to:

1. **Quantify element-level extraction performance** — For each of 14 clinical features, compare AI vs. human accuracy, sensitivity, specificity, PPV, NPV, and fabrication rate.
2. **Identify fragile elements** — Which features are most prone to AI fabrication or omission?
3. **Test statistical significance** — One-sided McNemar tests (H1: AI error > Human error) per element per metric.
4. **Model predictors of AI failure** — Use document-level text complexity, OCR quality, and embedding features to predict which cases/elements the AI will fail on.
5. **Evaluate prompt engineering impact** — Track how prompt versions and RAG configurations affect extraction quality over time.

## Key Questions

1. Does the AI fabrication rate significantly exceed the human fabrication rate for any clinical element?
2. Which document-level features (OCR quality, lexical diversity, negation frequency) best predict AI extraction errors?
3. Does RAG-based retrieval reduce hallucination compared to full-document prompting?
4. Are Radiology elements more or less fragile than Pathology elements for LLM extraction?
5. How do prompt iterations change diagnostic metrics across extraction runs?

## Dataset Description

The primary dataset contains **200 patient cases × 45 columns**:

| Variable Group | Count | Description |
|---|---|---|
| Source (ground truth) | 14 | Binary (0/1): feature present in source documents |
| Human annotator | 14 | Coded 1/2/3/N/A: human extraction status |
| AI annotator | 14 | Coded 1/2/3/N/A: LLM extraction status |
| Covariates | 2 | `tumor_invasive_dcis` (1=Invasive, 2=DCIS), `complex_case_status` (0/1) |
| Identifier | 1 | `surgeon_id` (de-identified, 20 unique) |

### Clinical Elements (14 features)

| # | Element | Domain |
|---|---|---|
| 1 | Lesion Size | Radiology |
| 2 | Lesion Laterality | Radiology |
| 3 | Lesion Location | Radiology |
| 4 | Calcifications / Asymmetry | Radiology |
| 5 | Additional Enhancement (MRI) | Radiology |
| 6 | Extent | Radiology |
| 7 | Accurate Clip Placement | Radiology |
| 8 | Workup Recommendation | Radiology |
| 9 | Lymph Node | Radiology |
| 10 | Chronology Preserved | Radiology |
| 11 | Biopsy Method | Radiology / Pathology |
| 12 | Invasive Component Size (Pathology) | Pathology |
| 13 | Histologic Diagnosis | Pathology |
| 14 | Receptor Status | Pathology |

### Annotator Coding Scheme

| Code | Meaning | Confusion Matrix Role |
|---|---|---|
| 1 | Correctly extracted (matches source) | TP (if source=1) |
| 2 | Omission (source present, not extracted) | FN |
| 3 | Fabrication (source absent, extracted anyway) | FP |
| N/A | Correctly not extracted (source absent) | TN |
